🔔 PR Slack Notification Setup

Complete guide to integrate GitHub Actions with Slack notifications

📋 Overview: This guide will help you set up automated Slack notifications for Pull Requests in your GitHub repository using GitHub Actions.

1 Copy and Merge the Workflow File

📁 File Location

Copy the workflow file to your repository's workflow directory:

.github/workflows/pr-slack-notification.yml

📝 Workflow File Details

File Name: pr-slack-notification.yml

Purpose: Automatically sends Slack notifications when PR events occur

Trigger Events:

  • opened: When a new PR is created
  • reopened: When a closed PR is reopened
  • ready_for_review: When a draft PR is marked ready
  • synchronize: When new commits are pushed to the PR
  • workflow_dispatch: Manual trigger for testing

🔧 How to Copy the File

  1. Create the .github/workflows/ directory in your repository if it doesn't exist: 
    mkdir -p .github/workflows
  2. Copy the pr-slack-notification.yml file to this directory
  3. If you already have workflows, ensure this file doesn't conflict with existing ones
  4. Commit and push the file to your repository: 
    git add .github/workflows/pr-slack-notification.yml
git commit -m "Add PR Slack notification workflow"
git push
⚠️ Important: The workflow file references ${{ secrets.SLACK_WEBHOOK_URL }} which must be configured in Step 2 before the workflow will function properly.

2 Slack Notification Setup

ℹ️ Note: Slack integration is used for PR notifications. This setup is required for the workflow to send messages to your Slack channel.

🔗 Create a Slack Webhook

  1. Go to https://api.slack.com/apps
  2. Click "Create New App"
  3. Select "From scratch"
  4. Enter an app name (e.g., "PR Notifications") and select your workspace
  5. Click "Create App"
  6. In the left sidebar, click "Incoming Webhooks"
  7. Toggle "Activate Incoming Webhooks" to On
  8. Scroll down and click "Add New Webhook to Workspace"
  9. Select the target channel where you want notifications to appear
  10. Click "Allow"
  11. Copy the webhook URL (it will look like: https://hooks.slack.com/services/T00000000/B00000000/XXXXXXXXXXXXXXXXXXXX)

💻 Local Slack Configuration (Optional)

If you're running the notification system locally for testing:

  1. Copy .env.example to a local .env file: 
    cp .env.example .env
  2. Add your webhook URL to the .env file: 
    SLACK_WEBHOOK_URL=https://hooks.slack.com/services/YOUR/WEBHOOK/URL
⚠️ Security: Never commit the .env file to your repository. Ensure it's listed in .gitignore.

🔐 GitHub Actions Slack Configuration

Add the webhook URL as a repository secret for GitHub Actions:

  1. Go to your GitHub repository
  2. Click Settings (repository settings, not your account)
  3. In the left sidebar, expand Secrets and variables
  4. Click Actions
  5. Click "New repository secret"
  6. Enter the following details:
    • Name: SLACK_WEBHOOK_URL
    • Value: Your Slack webhook URL
  7. Click "Add secret"
✅ Path Summary: GitHub Repository → Settings → Secrets and variables → Actions → New repository secret

3 Workflow Customization

🎨 Customization Options

The workflow file can be customized to fit your team's requirements:

Customization Location in File Description
Trigger Events Lines 3-6 Add/remove PR event types (e.g., closed, labeled)
Team Mention Line 71 Change @ebmc-gui-squad to your team's Slack handle
Notification Icons Lines 44-48 Customize emojis for different PR states
Message Format Lines 56-125 Modify the Slack Block Kit JSON structure
Draft PR Handling Line 52 Currently skips draft PRs; remove condition to include them

📝 Example Customizations

Change Team Mention:

"text": "*👥 @your-team-name, Please review*"

Add More PR Events:

on:
  pull_request:
    types: [opened, reopened, ready_for_review, synchronize, closed, labeled]

Include Draft PRs:

Remove the condition && github.event.pull_request.draft == false from lines 40 and 52

💡 Tip: You can test your workflow manually using the "workflow_dispatch" trigger. Go to Actions → PR Slack Notification → Run workflow.

🧪 Testing Your Setup

Manual Test

  1. Go to your repository on GitHub
  2. Click the Actions tab
  3. Select "PR Slack Notification" from the workflows list
  4. Click "Run workflow"
  5. Select the branch and click "Run workflow"
  6. Check your Slack channel for the test notification

Live Test

  1. Create a new branch in your repository
  2. Make a small change and commit it
  3. Open a Pull Request
  4. Check your Slack channel for the notification

🔍 Troubleshooting

Common Issues

❌ No notification received:

  • Verify the SLACK_WEBHOOK_URL secret is set correctly in GitHub
  • Check if the PR is a draft (drafts are skipped by default)
  • Review the Actions tab for workflow execution logs
  • Ensure the Slack app has permission to post in the channel

❌ Workflow not triggering:

  • Confirm the workflow file is in .github/workflows/
  • Check the file has a .yml or .yaml extension
  • Verify the YAML syntax is correct (no indentation errors)
  • Ensure the PR event type matches the configured triggers

❌ 400 Bad Request error:

  • Verify the webhook URL is correct and complete
  • Check if the Slack app is still installed in your workspace
  • Ensure the JSON payload format is valid

📚 Additional Information

Notification Content

Each PR notification includes:

  • 📊 Repository name
  • 👤 PR author
  • 📝 PR title
  • 🌿 Source and target branches
  • 🔗 Direct link to the PR
  • 📄 Link to view changed files

Notification Types

  • 🔔 New Pull Request: When a PR is first opened
  • ✅ PR Ready for Review: When a draft PR becomes ready
  • 🔄 PR Updated: When new commits are pushed
  • 🧪 Test Notification: Manual workflow dispatch
